**# Financial PDF Extractor

A Next.js 14 app that reads financial PDFs in the browser (PDF.js), sends extracted text to the OpenAI API for structured JSON, presents a human-review grid, and exports reviewed rows to **CSV** or **Excel (.xlsx)**.

## Features

- Local PDF text extraction in the browser (no upload of raw PDF bytes to OpenAI)
- Server-side **`POST /api/financial-extract`** for structured extraction via OpenAI
- Demo sample PDFs under `public/samples/` (optional Google Drive preview mode via env)
- Pipeline UI with upload, processing steps, review desk, and export toolbar

## Prerequisites

- **Node.js** 18+ (LTS recommended)
- **npm** (ships with Node)

## Quick start

```bash
git clone <repository-url>
cd financial-pdf-extractor
npm install
```

Copy environment template and add your API key:

```bash
copy .env.example .env.local   # Windows
# cp .env.example .env.local  # macOS / Linux
```

Edit **`.env.local`** and set at least **`OPENAI_API_KEY`**.

Start the development servers (Next.js + Tailwind watch):

```bash
npm run dev
```

Open **http://localhost:3040** in your browser (the dev script fixes this port).

If styles look wrong or the build cache is stale:

```bash
npm run dev:clean
```

## Production build

```bash
npm run build
npm start
```

By default **`npm start`** listens on port **3000**. Set **`PORT`** if you need another port.

## Environment variables

See **`.env.example`** for all documented variables.

| Variable | Required | Purpose |
|----------|----------|---------|
| `OPENAI_API_KEY` | Yes (for AI structuring) | Server-side OpenAI API key |
| `OPENAI_MODEL` | No | Model id (defaults are set in code if unset) |
| `NEXT_PUBLIC_USE_GOOGLE_DRIVE_SAMPLES` | No | Set to `true` only if you want Drive-based demo previews (bundled samples are the default) |

Never commit **`.env.local`** or real secrets.

## npm scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Runs Tailwind CLI watch + Next.js dev on port **3040** |
| `npm run dev:clean` | Deletes **`.next`**, then same as `dev` |
| `npm run build` | Normalizes line endings, emits minified Tailwind CSS, runs `next build` |
| `npm start` | Production server after `npm run build` |
| `npm run lint` | ESLint |
| `npm run normalize:eol` | Forces LF on key CSS/config files (Windows-friendly) |
| `npm run ensure-samples` | Ensures demo PDFs exist under `public/samples/` |

## Styling (Tailwind CSS)

Utilities are compiled with the **Tailwind CLI** into **`public/app-tailwind.css`** and loaded from the root layout. That avoids relying on Next.js webpack PostCSS alone (which can fail on some Windows setups). The file is generated by **`postinstall`**, **`prebuild`**, and the **`dev`** script; it is listed in **`.gitignore`**.

## Project layout (high level)

- **`app/`** — App Router pages, layout, global CSS source (`globals.css`), API routes
- **`components/`** — UI and workflow (`FinancialWorkflow`, `UploadDropzone`, `ReviewTable`, etc.)
- **`lib/`** — PDF text, detection, AI fetch/schema, export, validation
- **`public/samples/`** — Demo PDF assets (also populated by install scripts)
- **`scripts/`** — Sample PDFs, PDF.js worker copy, Tailwind emit, dev orchestration, LF normalization
